# Procedure Topic: Recruitment

## Slack-Integrated Recruitment Operations Framework (ROF)

- Project Context: Vector DB Research Project (Dr. Mussman)
- Initiative Manager: Dylan Bona
- Topic Area: Recruitment Process Standardization
---

## Overview

This initiative proposes the development of a modular, Slack-integrated Recruitment Operations Framework (ROF) to standardize recruitment, evaluation, and onboarding processes within the Vector DB Project. The goal is to reduce onboarding ambiguity, improve candidate–project alignment, and preserve institutional knowledge across semesters.

The framework will serve as a reusable recruitment template for other HAAG research projects.

---

## Standardized Procedures

Procedure 1.1 – Capacity & Skill Gap Assessment

- Identify number of recruits needed
- Define required technical skills
- Define expected weekly commitment
- Define deliverable expectations
- Output: Published role description

Procedure 2.1 – Standardized Application Form

Required fields:
- Academic background
- Relevant coursework
- Technical stack
- Availability
- Prior research experience
- Motivation statement

Applications submitted via: Slack Workflow Form

- Deliberation occurs in a determined private Slack Channel

Procedure 3.1 – Onboarding Packet

Includes:
- Project Overview
- Git Workflow/Access
- Documentation standards
- Operational Expectations
- Current Team Profile

## Slack Recruitment Architecture

Public Channels
- #vectordb-general
  - High level overview of Vector DB project through Slack Canvas
- #vectordb-recruitment
  - Standardized Application Form using Slack Workflow Form

Private Channels
- #vectordb-recruitment-review
  - Dedicated private slack channel for admins to review submitted forms
- #vectordb-onboarding
  - Channel where new recuits are provided Onboarding Packet
- #vectordb-admin
  - Private discussion channel for Project Recruitment Admin
